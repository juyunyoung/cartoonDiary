# PRD — Cartoon Diary  
**“일기를 쓰면, 하루가 4컷 만화가 된다”**

---

## 1. 제품 개요 (Overview)

### 1.1 제품명
**Cartoon Diary**

### 1.2 한 줄 설명
사용자가 작성한 일기를 AI가 이해하여 **4컷 만화로 자동 변환**해 주는 멀티모달 다이어리 앱

### 1.3 제품 비전
사람들은 기록하고 싶어 하지만, 글만으로는 오래 남기기 어렵다.  
Cartoon Diary는 **텍스트 기록을 시각적 기억으로 바꾸어**,  
일상을 “보관하고, 돌아보고, 공유하고 싶은 경험”으로 만든다.

---

## 2. 문제 정의 (Problem Statement)

### 2.1 사용자 문제
- 일기 쓰기는 귀찮고 지속하기 어렵다
- 텍스트만 남기면 감정과 장면이 흐릿해진다
- 회고하거나 공유할 만한 결과물이 잘 나오지 않는다

### 2.2 기존 해결책의 한계
- 텍스트 다이어리 앱 → 감정 몰입도 낮음
- 이미지 생성 앱 → 스토리 맥락 부족
- 만화 제작 툴 → 진입 장벽 높고 시간 소모 큼

---

## 3. 목표 (Goals)

### 3.1 제품 목표
- **3분 이내**에 “일기 → 4컷 만화” 결과 제공
- 글을 잘 못 써도 만화가 만들어지는 경험 제공
- 결과물이 **저장·공유하고 싶은 수준의 완성도**를 가질 것

### 3.2 해커톤 목표 (Amazon Nova Hackathon)
- Amazon Nova 모델을 핵심 기능에 실질적으로 활용
- Agentic AI 구조(Strands Agents + LangChain + MCP) 명확히 구현
- 멀티모달 이해 + 생성(텍스트 → 이미지) 파이프라인 증명

---

## 4. 타겟 사용자 (Target Users)

### 4.1 주요 타겟
- 일기·감정 기록을 하고 싶지만 꾸준히 못하는 사람
- SNS에 “오늘 하루”를 가볍게 공유하고 싶은 사용자
- AI 생성 콘텐츠에 친숙한 얼리어답터

### 4.2 사용 맥락
- 하루 마무리(잠들기 전)
- 인상 깊은 하루를 기록하고 싶을 때
- 여행, 이벤트, 특별한 날의 기록

---

## 5. 핵심 사용자 시나리오 (User Flow)

1. 사용자가 오늘의 일기를 작성
2. 스타일(귀여움 / 코미디 / 드라마 등) 선택
3. “만화로 만들기” 버튼 클릭
4. AI가 일기를 분석해 4컷 스토리보드 생성
5. 컷별 만화 이미지 생성
6. 4컷 만화 결과 확인
7. 필요 시 재생성 또는 저장/공유

---

## 6. 핵심 기능 요구사항 (Functional Requirements)

### 6.1 입력
- 텍스트 일기 (짧은 문장도 허용)
- 선택 옵션:
  - 오늘의 기분
  - 만화 톤/스타일
  - 주인공 이름(닉네임)

### 6.2 처리 로직 (AI 파이프라인)

#### 1) 스토리보드 생성
- Amazon Nova (Nova 2 Lite) 기반
- 일기를 다음 구조의 **4컷 JSON**으로 변환:
  - 컷 번호
  - 배경(장소/시간)
  - 등장인물
  - 행동(1문장)
  - 감정/표정
  - 대사 또는 내레이션
  - 화면 구도

#### 2) 에이전트 기반 검증
- 빈 컷, 논리 오류, 과도한 텍스트 검출
- 개인정보(이름, 연락처 등) 자동 마스킹

#### 3) 이미지 프롬프트 생성
- 캐릭터 외형/스타일은 고정
- 컷별 장면 요소만 변화

#### 4) 이미지 생성
- Amazon Nova Canvas (Bedrock)
- 컷별 이미지 생성 후 품질 검사
- 실패 컷만 자동 재생성

#### 5) 결과 합성
- 말풍선 + 자막 렌더링
- 4컷을 1장 이미지로 합성

---

## 7. 출력 (Outputs)

- 4컷 만화 이미지 (PNG/JPG)
- 컷별 스토리보드(JSON)
- 재생성 가능한 메타데이터

---

## 8. 비기능 요구사항 (Non-Functional Requirements)

### 8.1 성능
- 전체 생성 시간: 평균 1~2분 이내
- 비동기 처리 + 진행 상태 표시

### 8.2 일관성
- 동일 캐릭터/스타일 유지
- 컷 간 스토리 연결성 보장

### 8.3 보안/프라이버시
- HTTPS 통신
- 저장 데이터 암호화
- 기본 익명화 처리
- 사용자 요청 시 기록 삭제 가능

---

## 9. 기술 요구사항 (Technical Requirements)

### 9.1 필수 기술 (Hackathon Rule)
- Amazon Nova 모델 최소 1종 이상 사용
- Agentic AI 구성:
  - Strands Agents
  - LangChain
  - MCP Tools

### 9.2 기술 스택 (예시)
- Frontend: Web 또는 Mobile (React / Flutter)
- Backend: Python + FastAPI
- AI:
  - Nova 2 Lite (텍스트/스토리보드)
  - Nova Canvas (이미지 생성)
- Storage:
  - S3 (이미지)
  - DB (스토리보드/메타데이터)

---

## 10. 성공 지표 (Success Metrics)

- 첫 생성 성공률
- 재생성 버튼 사용 빈도
- 생성 결과 저장/공유 비율
- 데모 영상에서 “AI가 자동으로 개선하는 흐름” 명확성

---

## 11. 범위 제외 (Out of Scope – MVP 기준)

- 사용자 커스터마이징 캐릭터 에디터
- 장편 만화(4컷 초과)
- 소셜 피드 / 댓글 시스템
- 음성 일기(향후 확장)

---
